            ## Задача: Проектирование API для KYC
**Контекст:** Наша платформа расширяется и требует внедрения системы "Know Your Customer" (KYC) для верификации пользователей.
**Требования:**
1. Спроектировать RESTful API (описать эндпоинты, методы, DTO) для следующих операций:
    - Загрузка документов (паспорт, ID-карта).
    - Получение текущего статуса верификации.
    - Уведомление пользователя об изменении статуса (через webhook).
2. Описать схему хранения данных (например, в PostgreSQL).
3. Указать, как бы вы обеспечили безопасность загружаемых файлов и данных.
**Время выполнения:** 2 часа.
**Метрики успеха:** Полнота описания API, логичность схемы данных, продуманность вопросов безопасности.
